## Architecture générale

```

  rules.go          # toutes les règles OWASP
  rules_python.go   # règles spécifiques Python

  rules_go.go       # règles Go
  rules_shell.go    # règles Shell/Bash
  rules_sql.go      # règles SQL
  rules_secrets.go  # détection secrets/credentials
  fixer.go        # auto-correction via LLM
  reporter.go     # formatage des rapports

internal/security/sandbox/
  sandbox.go      # interface + factory
  linux.go       # Linux namespaces
  windows.go     # Windows Job Objects
  android.go    # Android detection + fallback
  workdir.go    # isolation workdir (fallback universel)
```

```
Critical = bloquer la livraison
High     = bloquer, corriger avant de continuer
Medium   = logguer, suggérer correction
```

## Partie 2 — Sandbox et Isolation

### Niveaux d'isolation

| Level | OS | Mécanisme |
|---|---|---|
| 2 | Linux | Namespaces (CLONE_NEWNET, CLONE_NEWPID, etc.) |
| 2 | Windows | Job Objects |
| 1 | Android | SELinux + ulimit |
| 1 | Fallback | Workdir isolation |

### Linux Namespaces

```go
Cloneflags: syscall.CLONE_NEWNET |
            syscall.CLONE_NEWPID |
            syscall.CLONE_NEWUTS |
            syscall.CLONE_NEWIPC |
            syscall.CLONE_NEWNS
```

### Windows Job Objects

- Memory limits
- CPU limits  
- Active process limits
- UI restrictions (clipboard, handles)

### Config

```go
type Config struct {
    MaxRAMMb       int
    MaxCPUSec      int
    MaxFileSizeMb   int
    MaxFileHandles int
    MaxProcesses   int
    AllowNetwork   bool
    WorkDir       string
    Timeout       time.Duration
}
```
