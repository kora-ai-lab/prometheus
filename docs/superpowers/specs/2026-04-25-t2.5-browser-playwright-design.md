## 1. Overview

Browser automation with Playwright as fallback when CDP is insufficient. Includes ADB for mobile apps and CAPTCHA handling.

---

## 2. Features

### 2.1 Playwright Integration (T2.5.1)

- Install via Capability Engine (playwright or puppeteer)
- PlaywrightClient implements BrowserClient interface

### 2.2 Client Interface (T2.5.2)

Existing interface in `client.go`:
```go
type Client interface {
    Navigate(url string) error
    GetHTML() (string, error)
    GetText() (string, error)
    Click(selector string) error
    Fill(selector, value string) error
    Submit(selector string) error
    Screenshot() ([]byte, error)
    WaitForSelector(selector string, timeout time.Duration) error
    WaitForNavigation(timeout time.Duration) error
    EvalJS(script string) (any, error)
    GetCookies() ([]*Cookie, error)
    SetCookie(c *Cookie) error
    ScrollDown(pixels int) error
    Close() error
}
```

### 2.3 ADB Mobile (T2.5.3)

- Android Debug Bridge for mobile-only apps
- Install via Capability Engine (adb)
- Commands: screenshot, tap, input text

### 2.4 CAPTCHA Handling (T2.5.4)

- Detect CAPTCHA elements
- Signal user with action=ask
- User provides solution

## 3. Architecture

```
internal/browser/
├── client.go        # existing - BrowserClient interface
├── cdp_client.go   # existing - stub (CDP not implemented)
├── playwright.go  # NEW - implements BrowserClient
└── adb.go         # NEW - Android Debug Bridge
```
