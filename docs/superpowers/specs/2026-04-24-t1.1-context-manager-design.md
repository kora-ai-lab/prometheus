## 1. Overview

Three-tier context management system for Prometheus:
- **Hot Buffer**: in-LLM messages (60-70% context window)
- **Warm Summary**: JSON-compacted conversation history
- **Cold Store**: SQLite persistence (Phase 1.2+)

## 2. Architecture

```go
type Manager struct {
    hotBuffer     []llm.Message  // recent messages in context
    warmSummary   string         // JSON compacted summary
    contextWindow int           // model's context window size
    threshold     float64        // compaction trigger ratio (0.60-0.80)
    keepLast      int            // messages to keep intact after compaction
    provider     llm.ModelProvider
}

type Snapshot struct {
    WarmSummary string         `json:"warm_summary"`
    HotBuffer   []llm.Message  `json:"hot_buffer"`
    ContextUsed int            `json:"context_used"`
}
```

## 3. Configuration

| Context Window | Threshold | KeepLast |
|----------------|-----------|----------|
| < 4K           | 0.60      | 5        |
| 4K - 8K        | 0.60      | 5        |
| 8K - 32K       | 0.65      | 10       |
| 32K - 128K     | 0.70      | 20       |
| > 128K         | 0.80      | 50       |

## 4. Core Methods

### 4.1 Add(msg)
- Appends message to hotBuffer
- Triggers compaction if `usageRatio() > threshold`

### 4.2 usageRatio() float64
- Returns: `(warmSummaryTokens + hotBufferTokens) / contextWindow`
- Uses `estimateTokens()` for fast token estimation

### 4.3 compact()
- Keeps last N messages intact
- Sends older messages to LLM with compaction prompt
- Stores result as `warmSummary` JSON
- Format:
```json
{
  "goal": "objectif principal",
  "done": ["accompli 1", "accompli 2"],
  "decisions": ["décision 1"],
  "errors_fixed": ["erreur→solution"],
  "state": "état actuel",
  "next": ["suite prévue"]
}
```

### 4.4 BuildMessages(systemPrompt) []llm.Message
```go
msgs := []llm.Message{{Role: "system", Content: systemPrompt}}
if m.warmSummary != "" {
    msgs = append(msgs, llm.Message{
        Role:    "system",
        Content: "[CONTEXTE PRÉCÉDENT]\n" + m.warmSummary,
    })
}
return append(msgs, m.hotBuffer...)
```

### 4.5 Snapshot() map[string]any
```go
return map[string]any{
    "warm_summary": m.warmSummary,
    "hot_buffer":   m.hotBuffer,
    "context_used": m.usageRatio() * m.contextWindow,
}
```

### 4.6 Restore(data map[string]any)
```go
if ws, ok := data["warm_summary"].(string); ok {
    m.warmSummary = ws
}
if hb, ok := data["hot_buffer"].([]any); ok {
    // convert []any to []llm.Message
    m.hotBuffer = convertMessages(hb)
}
```

## 5. Token Estimation

```go
func estimateTokens(s string) int {
    if s == "" { return 0 }
    // Empirical: 1 token ≈ 3.5 chars (Latin), × 1.15 safety margin
    chars := len([]rune(s))
    return int(float64(chars) / 3.5 * 1.15)
}
```

## 6. Persistence (per T1.1 analysis)

**Design Decision:** Persistent via task.Memory (not session-based)

- `Snapshot()` saves to `task.Memory["context_snapshot"]`
- Task stores to SQLite on each checkpoint (per T0.5.3 TODO)
- On reload: `Restore()` loads warmSummary + hotBuffer
- Result: Prometheus resumes exactly where it left off
