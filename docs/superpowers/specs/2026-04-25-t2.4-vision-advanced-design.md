## 1. Overview

Vision capabilities for Prometheus: automatic screenshot capture, PDF analysis, and mockup comparison. Uses the Capability Engine (T2.1) to install required tools (scrot, pdftoppm).

---

## 2. Features

### 2.1 Auto-Capture (T2.4.1)

**Triggers:**
- Web server started (port listening detected)
- HTML file created
- User explicitly requests screenshot

**Capture tools per OS:**
| OS | Tool | Command |
|---|------|---------|
| Linux (scrot) | `scrot` | `scrot -a '%t' screenshot.png` |
| macOS | `screencapture` | `screencapture -x screenshot.png` |
| Windows | PowerShell | `Add-Type -AssemblyName System.Windows.Forms; [System.Windows.Forms.Screen]::PrimaryScreen` |

### 2.2 PDF Analysis (T2.4.2)

**Flow:**
```
PDF → pdftoppm (installed via Capability Engine) → PNG pages → LLM analyze
```

**Implementation:**
- Check/install `poppler` via Capability Engine
- Convert: `pdftoppm -png -r 150 input.pdf output_prefix`
- Analyze each page with VisionProvider

### 2.3 Mockup Comparison (T2.4.3)

**Flow:**
```
mockup.png + result.png → compute diff → LLM analyze → iterate
```

**Features:**
- Pixel diff between mockup and result
- LLM identifies visual differences
- Automatic iteration (compare again after fixes)

## 4. Integration Points

### Task Loop Events

After certain task actions, trigger capture:

```go
// In task/loop.go
case action.Type == "exec" && isServerStart(result):
    deps.Capture.Trigger("server_started", port)

// After file creation
case action.Type == "create" && strings.HasSuffix(path, ".html"):
    deps.Capture.Trigger("html_created", path)

// On user request
case action.Type == "capture":
    deps.Capture.Trigger("explicit", "")
```

### Capability Engine

```go
// Install tools when needed
capEngine.Ensure(ctx, "scrot")      // Linux screenshot
capEngine.Ensure(ctx, "poppler")   // PDF tools
```
