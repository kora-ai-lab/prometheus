# T2.6 Self-Update Design

> Date: 2026-04-25
> Status: Approved
> Parent: Phase 2 (The Evolution)

## Goal

Implement self-update mechanism for Prometheus to automatically upgrade itself when a new version is available on GitHub Releases.

## Architecture

```
User runs /update command
        ↓
Fetch latest release from GitHub API
        ↓
Compare local vs latest version
        ↓
If newer → show "Version X.Y.Z available. Upgrade? (y/n)"
        ↓
User confirms → download binary + SHA256
        ↓
Verify checksum matches
        ↓
Replace binary, restart
```

## Components

### 1. Update Checker

- **File:** `internal/update/checker.go`
- **Methods:**
  - `CheckForUpdate() (hasUpdate bool, latestVersion string, err error)`
  - Fetches `https://api.github.com/repos/{owner}/{repo}/releases/latest`
  - Parses semver from `tag_name` (e.g., "v1.2.3" → "1.2.3")
  - Compares with current version using semver comparison

### 2. Downloader

- **File:** `internal/update/downloader.go`
- **Methods:**
  - `DownloadAndVerify(url, checksumURL string) (tmpPath string, err error)`
  - Downloads binary to temp directory
  - Fetches SHA256 from release assets or release body
  - Verifies hash matches
  - Returns path to verified binary

### 3. Upgrader

- **File:** `internal/update/upgrader.go`
- **Methods:**
  - `ApplyUpdate(tmpPath string) (err error)`
  - Replaces current binary with new one
  - Creates backup of old binary
  - Returns error if replace fails

### 4. CLI Integration

- **File:** `cmd/update.go`
- Adds `/update` command to CLI
- Orchestrates: check → prompt → download → verify → apply

## Algorithm

### Check Version

```
1. Call GitHub API: GET https://api.github.com/repos/{owner}/{repo}/releases/latest
2. Parse tag_name → latestVersion (strip "v" prefix)
3. Parse current version from build info or constant
4. Compare: semver Compare(local, latest)
5. If latest > local → return hasUpdate=true
```

### Download & Verify

```
1. Find binary asset (os/arch matching current system)
2. Download to $TMPDIR/prometheus-{version}
3. Extract SHA256 from:
   a. Release body text containing "SHA256: {hash}"
   b. Or separate .sha256 asset
4. Compute hash of downloaded file
5. If hash != expected → delete and error
6. Return tmpPath
```

### Apply Update

```
1. Create backup: cp currentBin currentBin.backup
2. Copy newBin → currentBin location
3. Make executable: chmod +x currentBin
4. Fork+exec to restart with same args
```

## Error Handling

| Scenario | Behavior |
|----------|----------|
| GitHub unreachable | Return error "cannot check for updates: network error" |
| No releases | Return error "no releases found" |
| Verify fails | Delete downloaded, return error "checksum mismatch" |
| Replace fails | Keep backup, return error "upgrade failed" |

## Configuration

Add to `prometheus.toml`:

```toml
[update]
enabled = true          # Enable self-update (default: true)
owner = "anomalyco"   # GitHub owner (default: "anomalyco")
repo = "prometheus"  # GitHub repo (default: "prometheus")
checkOnStartup = true # Check on startup (default: true)
```
