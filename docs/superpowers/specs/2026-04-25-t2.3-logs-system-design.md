## 1. Overview

Log system with three layers:
- **Raw logs** — for machines (`.jsonl`, `.jsonl.zst`)
- **Summaries** — for humans/LLM (`.md`)
- **FTS5 index** — for fast queries

Philosophy: **Brute logs for machines, summaries for humans, FTS5 for queries.** These three layers never duplicate — never search `.jsonl.zst`, search the index.

## 2. Directory Structure

```
~/.prometheus/
  logs/
    2025-01-15.jsonl          ← active day (real-time write)
    2025-01-14.jsonl.zst      ← yesterday, compressed
  summaries/
    2025-01-15.md             ← daily summary
    2025-01-14.md
  archive/
    2025-01/
      2025-01-01.jsonl.zst    ← archived raw logs
      summary.md              ← monthly summary
    2024-12/
  prometheus.db              ← SQLite with FTS5
```

## 3. Layer 1 — Logger (existing + enhancement)

Existing `logger.go` enhanced with:

- **Daily rotation** — new file each day, format `YYYY-MM-DD.jsonl`
- **Async writes** — buffered channel, flush every 100 entries or 1s
- **Redaction** — `RedactSecrets()` called before each write

**Log Entry Format (JSON Lines):**
```json
{"ts":"2025-01-15T09:00:00.123Z","session":"sess_abc","task_id":"task_xyz","level":"exec","event":{"type":"exec","command":"mkdir -p dir","exit_code":0,"duration_ms":23}}
```

**What is NEVER logged:** credentials, file content > 100 bytes, full stdout/stderr (only lengths), raw LLM responses.

## 4. Layer 2 — Compressor

**Trigger:** At Prometheus startup, compress old `.jsonl` files from before today.

```go
// internal/logging/compressor.go

type Compressor struct {
    logsDir string
    level  zstd.EncoderLevel
}

// CompressOldLogs runs in background goroutine at startup
func (c *Compressor) CompressOldLogs(ctx context.Context) error
func (c *Compressor) CompressFile(src, dst string) error
func (c *Compressor) VerifyIntegrity(path string) bool
```

**Compression:**
- Use `klauspost/compress/zstd` (pure Go)
- Level: `zstd.SpeedBestCompression`
- Expected gain: **5MB → ~500KB (8-12%)**

## 5. Layer 3 — Summarizer

**Trigger:** Once per day, at first session next day or via CLI.

```go
// internal/logging/summarizer.go

type Summarizer struct {
    logsDir    string
    summaryDir string
    provider  llm.Provider
}

func (s *Summarizer) SummarizeDay(ctx context.Context, date string) error
func (s *Summarizer) buildDigest(events []LogEntry) string
func (s *Summarizer) generateSummary(ctx context.Context, date, digest string) (string, error)
```

**Process:**
1. Load log events for the day (compressed or not)
2. Build digest (<2000 tokens): session goals, files created, failed commands, stats
3. Call LLM with summary prompt
4. Write `.md` to summaries/
5. Index in FTS5

**Summary Format:**
```markdown
<!-- prometheus_summary_version:1 date:2025-01-15 generated:2025-01-16T08:00:00Z -->
# Journal 2025-01-15

## En bref
[2-3 phrases summarizing the day]

## Accompli
[bullet list]

## Difficultés
[bullet list]

## Stats
Durée: X | Commandes: N | Tokens: N
```

**Process:**

2. Generate monthly summary from daily summaries
3. Write `summary.md` to archive/

## 7. Layer 5 — Query + FTS5

### Temporal Query Parser

```go
// internal/logging/query.go

type TimeQuery struct {
    StartDate string
    EndDate  string
    Keywords []string
}

func ParseTemporalQuery(q string) *TimeQuery
```

**Supported patterns:**
- `hier` → yesterday
- `lundi dernier` → last Monday
- `semaine dernière` → Monday→Sunday last week
- `mois dernier` → previous month
- `aujourd'hui` → today
- `il y a N jours` → N days ago

### FTS5 Search

**SQL (migration):**
```sql
CREATE TABLE IF NOT EXISTS log_index (
    date       TEXT NOT NULL,
    month     TEXT NOT NULL,
    summary_md TEXT,
    projects  TEXT,
    technologies TEXT,
    goals     TEXT,
    indexed_at DATETIME
);

CREATE VIRTUAL TABLE IF NOT EXISTS log_search
USING fts5(date, summary_md, projects, technologies, goals,
           content='log_index', content_rowid='rowid');
```

**Search:**
```go
func (s *Store) SearchLogs(tq *TimeQuery) ([]LogIndexEntry, error)
func (s *Store) IndexSummary(date, summaryMd string) error
```

## 9. Integration

Logger injected into `TaskDeps`, called at key points:

```go
// After each exec
deps.Logger.LogExec(t.ID, result)

// After each LLM call
deps.Logger.LogLLM(t.ID, resp, latency, action)

// After file creation
deps.Logger.LogFileCreated(t.ID, path, size)

// On blocking
deps.Logger.LogBlocked(t.ID, reason)

// On compaction
deps.Logger.LogCompaction(t.ID, before, after, tokens)
```
