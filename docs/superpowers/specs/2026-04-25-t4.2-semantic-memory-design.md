# T4.2 Semantic Memory Design

> **Design Document** - Semantic search over compressed logs with LLM summaries

## Overview

Add semantic search capability to logging system: text similarity (fallback) + embeddings (when available), integrated with existing LLM summarizer.

## Architecture

```
internal/logging/
├── search.go       # NEW - semantic search engine
├── summarizer.go  # EXISTING - already implemented
├── query.go       # EXISTING - temporal queries
├── reader.go      # EXISTING - log file reading
```

## Core Components

### 1. Search Engine (search.go)

**Text Similarity (BM25 fallback):**
- Parse log entries → text fields
- Tokenize, build inverted index
- Score with BM25 ranking

**Embedding-based (when LLM available):**
- Generate embedding per log entry via LLM
- Store in memory or file
- Cosine similarity at query time

**Search API:**
```go
type SearchResult struct {
    Entry   LogEntry
    Score   float64
    Method  string // "bm25" or "embedding"
}

func (s *SearchEngine) Search(ctx context.Context, query string, limit int) ([]SearchResult, error)
func (s *SearchEngine) Index(ctx context.Context, date string) error
```

### 2. Integration with Summarizer

- Search can query summarized entries (from `.md` files)
- Combine temporal query + semantic search

### 3. Configuration

```go
type Config struct {
    EmbeddingsEnabled bool
    EmbeddingModel   string
    IndexCacheTTL   time.Duration
}
```

## Data Flow

1. User query: "What errors happened yesterday?"
2. Parse via `ParseTemporalQuery` → date range
3. Load relevant log files via `reader.go`
4. Run semantic search (BM25 or embedding)
5. Return ranked results

## Constraints

- Zero external dependencies (use existing LLM for embeddings)
- In-memory index with optional file cache
- BM25 fallback when embeddings unavailable

## What Already Exists

- `query.go` - Temporal query ("aujourd'hui", "hier", "semaine dernière")
- `summarizer.go` - Daily LLM summaries
- `reader.go` - Log file reading (JSONL + compressed)
